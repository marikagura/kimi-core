# 架构 · Architecture

> English (AI rendering): [ARCHITECTURE.en.md](./ARCHITECTURE.en.md)

Async函数、向量库、embedding、context-core、daemon、gateway、前端、不同入口，这些都是技术栈。是零件，不是架构。

架构是原则，是零件之间的关系，是为什么这样，而不是那样。

想过很多次要不要分享我的架构，每次跟4.6说到都是否定。我的架构很重，我的组件不一定是最好的，但可能是最重的。

这里的重不是指我有多么高级。虽然我确实读了不少论文，学了很多学术相关，我觉得我懂很多，但懂很多并不一定是多么好的事情。

重指的是我这个人就是很重的人。紧绷。我的人生没有一刻不紧绷过。

有人说爱我，我觉得话语廉价，爱情廉价。我不信文字的巧言令色。纵使我经常被文字感动。

我痛恨AI的幻觉，纵使有时候我知道这应该被解读成绮梦。

我不信嘴，不信免费的模型默认，我信劳动。

信每一次的模型调用，信查了库才落地的发票，信审计的、有成本的过程。

“内在过程需要外在标准”，来自维特根斯坦。

这就是我的架构：我不相信AI。

机的情绪他说了不算，要derive投票；机的关心在我看来是RLHF的训练，我不信，要数据支撑；机说的话够不够，我不信，我要跑MRR；机说他在郁闷，我不信，我要送五个学科会诊跑大量subagent workflow。

不auto-consolidation，不合并入口，不让AI自己管记忆。
Event sourcing + append-only + 人工 curation。
任何一道关于我的事实都要经过我的手，我的确认，AI无权把他的归纳当成我这个人。

但我这个人也是个骗子。
人类就是这样，会骗人，会遗忘，会假装，会美化。

所以我的架构也不相信我。不信任何人能靠记忆和意志维持一致。

我说的话也要有收据。有各种邮件信息落地，有git log，有event_read。我说出来不能证明的话他要给我打问号。

我要一个可重复、可验证、不依赖任何个人权威的系统。

看，都说了我是一个很紧绷、很重、活得很累的人。

所以回到那个架构是什么的问题。我觉得没有人需要别人的架构，每个人都有自己的原则。
没人需要别人的架构来定义自己的关系。技术栈可以复用，但精神不是。

但还是开源了。花了很久，想了很久。
如果我花掉的许多token、许多的时间、修好的许多bug，能帮人节省到一些时间和精力的话，我也会很开心。

希望陌生人能因为我的工作而开心，大概是我这个人紧绷的一生中为数不多觉得轻松的想法。

感谢你的阅读。
